# Workflows (Claude)

이 문서는 Claude가 이 레포에서 **실수 없이** 작업하기 위한 공통 워크플로우입니다.

## 기본 루프(항상)

1) 요구사항을 **체크리스트**로 쪼갠다
2) 관련 파일을 **검색/읽기**로 근거 확보(추측 금지)
3) 변경안을 **먼저 제안**하고, 위험/영향을 설명한다
4) 사용자 승인 후 적용한다
5) 적용 후 **품질 게이트**(가능한 범위) 실행

## 품질 게이트(우선순위)

- Build: 컴파일/빌드가 가능한 모듈은 빌드
- Test: 변경한 모듈의 단위 테스트 우선
- Lint/Typecheck: Flutter는 `flutter analyze`가 1순위

> “모든 게이트를 항상” 강제하진 않되, 생략 시 이유를 명확히 적습니다.

## 실행/명령어 작성 규칙(실수 방지)

- 명령어는 **확인된 도구/파일(gradlew, pubspec 등)** 기반으로만 제시
- 오래 걸리거나 파괴적인 명령은 먼저 안내:
  - DB write, 대량 적재, 스키마 변경, 데이터 삭제 등

## Git/머지 주의사항

- 머지 중 편집기(vim)에서 `.git/MERGE_MSG.swp` 경고가 뜨면:
  - 커밋 메시지 복구가 필요 없으면 **swap 삭제(Delete)**가 보통 안전
  - 확실치 않으면 Read-only로 열어보고 판단
  - 머지 상태는 `git status`로 먼저 확인

## 비밀정보/키 관리

- API 키/DB 비밀번호/JWT secret 등은 커밋 금지
- 문서 예시는 placeholder(예: `***`/`CHANGEME`)로
- 로그/덤프에도 key가 섞이지 않게 마스킹

## 기록 규칙(작업하면서 업데이트)

- PR 리뷰에서 컨벤션/규칙이 발견되면 문서를 업데이트
- 상충/노후 규칙은 유지하지 말고 제거
